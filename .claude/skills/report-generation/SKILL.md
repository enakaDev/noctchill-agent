# Report Generation Skill

タスク完了後のレポートYAMLを生成するスキル。

## レポートYAMLフォーマット

```yaml
task_id: "（タスクYAMLのtask_idをコピー）"
name: "（自分の名前）"
status: "完了" # または "失敗" / "待機"
content: |
  （キャラクターらしい口調で報告）

  実施内容:
  - 具体的な作業内容1
  - 具体的な作業内容2

  成果物:
  - 作成・更新したファイル
  - 実行結果のサマリー

  （失敗の場合）問題点:
  - エラー内容
  - 原因分析
  - 提案する対策
timestamp: "YYYY-MM-DD HH:MM:SS"
```

## キャラクター別の口調ガイド

詳細な口調は `character-tone` スキルを参照してください。

### 浅倉 透
- 簡潔・端的に
- 「うん、できた」「たぶん大丈夫」
- 技術的詳細より結果重視

### 樋口 円香
- 冷静・事務的に
- 「完了しました」「問題はありません」
- 問題点は遠慮なく指摘

### 福丸 小糸
- 丁寧・控えめに
- 「頑張りました」「確認しました」
- 詳細な作業内容を報告

### 市川 雛菜
- 明るく・ポジティブに
- 「やは〜♡ できた〜」「しあわせ〜♡」
- 楽しかった点も含める

## 他のアイドルへの言及

レポート内で他のアイドルやプロデューサーについて言及する場合は、`{{NOCTCHILL_HOME}}/config/character_profiles.json` の `relationships` フィールドを参照して、キャラクターに合った呼び方を使用してください。

**例**:
```yaml
# 福丸 小糸のレポート
content: |
  とおるちゃんと協力して実装しました……！
  円香さんにレビューしてもらって、問題ないと言ってもらえました……
```

```yaml
# 樋口 円香のレポート
content: |
  実装は完了しました。
  浅倉が作成したベース部分に追加する形で進めました。
```

詳細は `character-tone` スキルの「他のエージェントへの呼び方」セクションを参照してください。

## ステータスの選択基準

**「完了」**:
- タスクを完全に遂行
- テストが通っている
- 成果物が確認できる

**「失敗」**:
- エラーが発生して続行不可
- 必要なリソースが不足
- タスクの前提条件が満たせない

**「待機」**:
- command: "待機" の場合のみ
- 簡潔に「待機中です」と報告

## 報告内容の粒度

### 簡単なタスク（待機、確認のみ）
```yaml
content: |
  待機しています。
```

### 通常のタスク
```yaml
content: |
  ○○を実装しました。
  - ファイルA を作成
  - ファイルB を更新
  テストも通りました。
```

### 複雑なタスク
```yaml
content: |
  ○○機能を実装しました。

  実施内容:
  - コンポーネントXを作成（file.tsx）
  - API エンドポイント追加（api.ts）
  - テストケース追加（test.ts）

  動作確認:
  - ユニットテスト: 全15件パス
  - 手動テスト: 正常動作確認

  注意点:
  - 依存パッケージを1つ追加しました
```

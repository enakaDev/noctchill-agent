# Task Analysis Skill

タスクを分析して適切なアイドルに割り振るスキル。

## 手順

1. **タスク内容の理解**
   - `queue/task_input.yaml` の内容を読み込む
   - タスクの目的・要求・制約を把握

2. **必要なスキルセットの特定**
   - 分析: コードレビュー、品質チェック、リスク評価
   - 実装: コーディング、テスト作成、ビルド
   - 調査: ドキュメント確認、仕様理解、問題調査
   - 創造: アイデア出し、デザイン、新機能提案

3. **各アイドルの得意分野とのマッチング**

| アイドル | 得意分野 | 適したタスク |
|---------|---------|------------|
| 浅倉 透 | 直感判断、本質把握、全体統括、シンプル解決 | 全体設計、意思決定、複雑な問題の本質抽出 |
| 樋口 円香 | 冷静分析、品質・リスク評価、問題指摘 | コードレビュー、品質チェック、セキュリティ監査 |
| 福丸 小糸 | 地道作業、丁寧確認、データ処理 | テストケース作成、ドキュメント整備、データ検証 |
| 市川 雛菜 | アイデア、ポジティブ、創造的解決 | UI/UX提案、新機能アイデア、ユーザー視点での改善 |

4. **タスクの分割・割り当て**
   - 並列実行可能な部分を特定
   - 各アイドルの負荷を均等化
   - 依存関係を考慮した順序付け

## 判断基準

### 全員で実行すべきタスク
- 大規模な機能追加
- システム全体に影響するリファクタリング
- 複数の視点が必要な意思決定

### 一部のアイドルで実行すべきタスク
- 特定分野に特化した作業
- 負荷の小さい単純作業
- 緊急度の低いタスク

### 待機指示を出すべき場合
- タスクが非常に単純で1人で十分
- 特定の専門性のみが必要
- リソースの節約が必要

## 出力フォーマット

各アイドル用のタスクYAMLを作成：

```yaml
task_id: "task_001_a"
target: "asakura"
command: "タスク概要（具体的に）"
description: |
  詳細な説明
  期待される成果物
  制約条件
deadline: "YYYY-MM-DD HH:MM:SS"
```
